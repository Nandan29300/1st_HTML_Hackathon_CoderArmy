1.) File Paths

A file path tells the browser where to find a file (HTML, CSS, JS, images).

Example:

<img src="images/logo.png" alt="Logo">

Output:

Displays an image called logo.png from the images folder.

2.) Absolute vs Relative Paths

Scenario 1: Linking to a file in the SAME folder

You are in index.html and want to link to about.html (both in the same folder).

Scenario 2: Linking to a file in a SUB-FOLDER (Going Down)

You are in index.html and want to display logo.png in the images folder.

<!-- Inside index.html -->
<img src="images/logo.png" alt="My Website Logo">

Scenario 3: Linking to a file in a PARENT FOLDER (Going Up)

You are in contact.html (inside pages/) and want to display logo.png (inside images/).

<!-- Inside pages/contact.html -->
<img src="../images/logo.png" alt="My Website Logo">

Explanation:

  • ../ takes you from pages/ up to my-website/
  • images/logo.png then takes you down into images/ folder to find the file

B) Absolute File Paths

An absolute file path gives the full, complete URL to a resource on the web. It starts with http:// or https://.

When to use it: Only for resources not on your own website.

<!-- External website -->
<a href="https://www.google.com">Search on Google</a>

Search on Google


<!-- Image hosted externally -->
<img src="https://upload.wikimedia.org/wikipedia/commons/3/38/HTML5_Badge.svg" alt="HTML5 Logo">

HTML5 SVG Logo

The Critical "Why": Why not use absolute paths for your own files?

Copying the full path from your computer (like C:/Users/Nandan/Desktop/my-website/images/logo.png) is a major mistake. It only works on your computer. Once uploaded to a server, it will break.

Browser Sandbox Explanation

  • The Inmates: HTML, CSS, JavaScript running inside the browser
  • The Prison Walls: The browser itself

The browser prevents code from freely accessing your computer‚Äôs file system for security reasons.

Core Problem: Malicious Website

Without this sandbox, a malicious website could try to read private files:

<!-- Malicious example -->
<img src="C:/Users/Nandan/Documents/MyPrivatePasswords.txt">

Summary

  1. For Your Own Files: ALWAYS use relative paths
    • filename.html (same folder)
    • folder/filename.html (go down)
    • ../filename.html (go up)
  2. For Other Websites‚Äô Files: ALWAYS use absolute paths
    • https://www.example.com/page.html

3.) HTML Boilerplate

The boilerplate is the basic structure every HTML page should have. Let's break it down line by line:

Example:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Document</title>
</head>
<body>
  
</body>
</html>

Line by Line Explanation:

  1. <!DOCTYPE html>
    Tells the browser to use HTML5 and standard-compliant mode, avoiding old "quirks mode". Must always be first.
  2. <html lang="en"> ... </html>
    Root element of the page. lang="en" helps search engines and screen readers understand the language.
  3. <head> ... </head>
    Contains metadata, links, and scripts that are not displayed directly on the page.
  4. <meta charset="UTF-8">
    Specifies the character set. UTF-8 supports almost all languages and symbols.
  5. <meta name="viewport" content="width=device-width, initial-scale=1">
    Makes the page responsive on different devices by matching the screen width and setting initial zoom to 100%.
  6. <title>...</title>
    Sets the page title for browser tabs, bookmarks, and search engines.
  7. <body> ... </body>
    The visible content of your page‚Äîheadings, paragraphs, images, links, etc.‚Äîgoes here.

But, the Website Works Even Without Boilerplate

Browsers are built to be extremely forgiving. They try to render something even if HTML is incomplete or missing tags.

What Browsers Do with "Bad" HTML:

  1. No <!DOCTYPE html>? Browser enters Quirks Mode to mimic old non-standard behavior.
  2. No <html> or <body> tags? Browser automatically generates them internally to create a DOM structure.
  3. No <meta charset="UTF-8">? Browser guesses character encoding. Special characters may break.
  4. No <title>? Browser uses the filename as the default tab title.

Why Use a Boilerplate Anyway?

  • Predictable Rendering: Avoid quirks mode; CSS works consistently across browsers.
  • Proper Characters: UTF-8 prevents broken symbols like ‚Ç¨, ‚Äî, ‚Äô.
  • Mobile-Friendly: Viewport meta ensures a good experience on phones and tablets.
  • SEO & Accessibility: Tags like lang and title improve search engine ranking and help screen readers.

Conclusion: You can build a page without boilerplate and it might appear fine, but a proper HTML boilerplate is the foundation for a professional, reliable, and future-proof website. Think of it as building a house on solid ground instead of on sand.

Tip: In VS Code, type ! and hit enter to generate the ready-made HTML boilerplate snippet automatically.

4.) Linking CSS & JS

You can add external CSS and JS files using <link> and <script> tags.

Example:

<!-- Link CSS -->
<link rel="stylesheet" href="css/style.css">

<!-- Link JS -->
<script src="js/script.js"></script>

5.) Multipage Website & Containers

Multipage Website

Break content into separate pages (home, about, contact). Use a consistent header, footer, and navigation for cohesion.

<nav>
  <a href="index.html">Home</a>
  <a href="about.html">About</a>
  <a href="contact.html">Contact</a>
</nav>

<div> - The Generic Box

Use <div> to group elements as a single unit:

<div class="profile-card">
    <img src="avatar.png">
    <h2>Nandan Aghera</h2>
    <p>Loves to code and teach HTML.</p>
</div>

class & id - Labels for Boxes

id: unique identifier, only one per page.
class: reusable label, can be used multiple times.

<div id="featured-profile" class="profile-card">...</div>
<div class="profile-card">...</div>

CSS targeting:

ID's are targeted by using the # and Classes are targeted by using .

 
#featured-profile { border: 2px solid gold; }
.profile-card { background: #eee; }

<span> - Inline Highlighter

Use <span> to style text within a line without breaking layout:

<p>This is very <span class="highlight">important</span> information.</p>

Semantic Tags

  • <header>: Top section with logo and navigation
  • <footer>: Bottom section with copyright info
  • <main>: Main content unique to this page (one per page)

6.) Common File Path Errors

  • Incorrect folder name or typo
  • Wrong relative path (../ or ./ mistakes)
  • Forgetting file extension (.css, .js, .png)
  • Case sensitivity on some servers (Logo.png ‚â† logo.png)

7.) üìñ Glossary of Terms

  1. File Path: Location of a file on a computer or server.
  2. Absolute Path: Full path including domain/drive.
  3. Relative Path: Path relative to current file.
  4. Boilerplate: Basic HTML structure for all pages.
  5. <link>: Tag to include CSS files.
  6. <script>: Tag to include JS files.
  7. Common Errors: Mistakes in file paths that prevent files from loading.
