1.) Forms: The Web Needs to Be a Two-Way Street

A website isn't just for reading. Forms let the user send information to the server, which is essential for logging in, searching, buying, posting comments, or messaging.

Core Problem: How do we reliably display fields, collect data, send it to the server, and specify how it is sent?

Solution: Use the HTML <form> element as the container for user input.

2.) Text Inputs, Labels, and IDs

The <input> element creates an input field. The type="text" attribute makes it a standard text box.

<input type="text">

Adding a Label

Labels explain the input‚Äôs purpose. Use <label> and connect it to the input using for and id.

<label for="firstName">First Name:</label>
<input type="text" id="firstName" name="firstName">

Clicking the label focuses the input; screen readers announce the label with the field.

3.) Radio Buttons & Checkboxes

Radio Buttons (Select One)

Used when the user can choose only one option. All buttons in the same group share the same name.

Sample Code is shown below:


<form>
  <label>T-Shirt Size:</label><br>
  <input type="radio" id="sizeS" name="shirtSize" value="small">
  <label for="sizeS">Small</label><br>

  <input type="radio" id="sizeM" name="shirtSize" value="medium">
  <label for="sizeM">Medium</label><br>

  <input type="radio" id="sizeL" name="shirtSize" value="large">
  <label for="sizeL">Large</label><br><br>
</form>

Checkboxes (Select Many)

Used when the user can select multiple options. Checkboxes in the same category share the same name but different value.

Sample Code is shown below:


<form>
  <label>Pizza Toppings:</label><br>
  <input type="checkbox" id="toppingPep" name="toppings" value="pepperoni">
  <label for="toppingPep">Pepperoni</label><br>

  <input type="checkbox" id="toppingMush" name="toppings" value="mushrooms">
  <label for="toppingMush">Mushrooms</label><br>

  <input type="checkbox" id="toppingOni" name="toppings" value="onions">
  <label for="toppingOni">Onions</label><br><br>

4.) Textarea & Dropdown Select

Textarea for Multi-Line Input

Use <textarea> for long text input like comments or addresses.

Sampel Code is shown below:


<form>
  <label for="comments">Any special instructions?</label><br>
  <textarea id="comments" name="comments" rows="4" cols="50" placeholder="Write here..."></textarea>
</form>

Dropdown Select

Use <select> for long lists, with individual <option> items.

Sample code is shown below:


<form>
  <label for="country">Country:</label><br>
  <select id="country" name="country">
    <optgroup label="Asia">
      <option value="in">India</option>
      <option value="cn">China</option>
    </optgroup>
    <optgroup label="Europe">
      <option value="uk">United Kingdom</option>
      <option value="fr">France</option>
    </optgroup>
    <optgroup label="America">
      <option value="us">USA</option>
      <option value="ca">Canada</option>
    </optgroup>
  </select>
</form>

8.)üìñ Glossary of Terms

  1. <form>: Container element that collects user input and sends it to the server.
  2. <input>: Used to create different types of input fields (text, password, number, etc.).
  3. <label>: Describes the purpose of an input field and improves accessibility.
  4. id / for: id uniquely identifies an element. The for attribute in <label> links to the matching input id.
  5. Radio Buttons: Allow the user to select only one option from a group (same name).
  6. Checkboxes: Allow the user to select multiple options.
  7. <textarea>: Multi-line text input for comments, addresses, or longer messages.
  8. <select> / <option>: Dropdown menu to choose from a list of options. <optgroup> groups related options.
  9. <fieldset> / <legend>: Group related inputs together with a title for better structure.
  10. required: An attribute that forces the user to fill in the field before submission.
  11. <button type="submit">: Submits the form. More flexible than <input type="submit">.
